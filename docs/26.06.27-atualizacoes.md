## 2026-06-27 — Auditoria Next.js Best Practices

### Contexto

Análise completa do código implementado confrontada com os princípios da skill `nextjs-best-practices` (Server Components, data fetching, routing patterns, metadata, error boundaries).

---

### 🔴 Correções Críticas

#### 1. `StudyCard.tsx` — Removido `'use client'` desnecessário

**Arquivo:** `src/app/components/StudyCard.tsx`

**Problema:** O componente estava marcado como `'use client'` sem usar nenhum hook (`useState`, `useEffect`) nem manipuladores de eventos. Isso enviava o componente desnecessariamente para o bundle do cliente.

**Correção:**

```diff
- 'use client';
-
  import Link from 'next/link';
```

**Impacto:** `StudyCard` agora é renderizado no servidor junto com `page.tsx`, reduzindo o bundle do cliente.

---

#### 2. `MarkdownRenderer.tsx` — Adicionado `'use client'` obrigatório

**Arquivo:** `src/app/components/MarkdownRenderer.tsx`

**Problema:** `react-markdown` usa APIs de browser internamente. Sem a diretiva, o Next.js tentaria executá-lo no servidor, causando potenciais erros de hidratação.

**Correção:**

```diff
+ 'use client';
+
  import ReactMarkdown from 'react-markdown';
```

### 🟡 Correções Médias

#### 3. Criados `loading.tsx` e `error.tsx` para ambas as rotas

**Arquivos criados:**

| Arquivo                             | Propósito                                     |
| ----------------------------------- | --------------------------------------------- |
| `src/app/loading.tsx`               | Skeleton animado (shimmer) para a home        |
| `src/app/error.tsx`                 | Error boundary global com botão de retry      |
| `src/app/estudo/[slug]/loading.tsx` | Skeleton para página de estudo individual     |
| `src/app/estudo/[slug]/error.tsx`   | Error boundary com link de volta à biblioteca |

**Motivação:** A skill aponta explicitamente como anti-pattern ignorar estados de carregamento e error boundaries. O Next.js App Router provê arquivos de convenção (`loading.tsx`, `error.tsx`) para isso.

**Estilos adicionados em `globals.css`:**

- Animação `@keyframes shimmer` para os skeletons
- Classes `.skeleton`, `.skeleton-card`, `.skeleton-*` para os placeholders
- Classes `.error-page`, `.error-retry-btn`, `.error-actions` para as páginas de erro

---

#### 4. `generateMetadata` — Adicionado Open Graph completo

**Arquivo:** `src/app/estudo/[slug]/page.tsx`

**Antes:** Retornava apenas `title` e `description`.

**Depois:**

```ts
return {
  title: `${title} | Estudos Bíblicos`,
  description,
  openGraph: {
    title,
    description,
    type: 'article',
    ...(image && { images: [{ url: image, alt: title }] }),
    ...(author && { authors: [author] }),
    ...(category && { section: category })
  }
}
```

**Impacto:** Melhora o compartilhamento em redes sociais e atende ao requisito da skill de incluir Open Graph images e metadados de artigo.

---

#### 5. `markdown.ts` — Eliminada duplicação de código

**Arquivo:** `src/lib/markdown.ts`

**Problema:** A lógica de normalização de tags e derivação de slug estava copiada e colada em `getStudies()` e `getStudyBySlug()`.

**Correção:** Extraídas duas funções auxiliares privadas:

```ts
// Normaliza metadados (garante tags como array)
function parseMetadata(data): MarkdownMetadata { ... }

// Deriva slug limpo a partir do nome do arquivo
function fileToSlug(filename: string): string { ... }
```

`getStudies()` refatorado para usar `.filter().map()` ao invés de `for...of` com `push`.

`getStudyBySlug()` simplificado de 35 linhas para 10 linhas usando `.find()`.

### 🟠 Correções Menores

#### 6. `StudyCard.tsx` — Corrigido `key={index}` para `key={tag}`

```diff
- <span key={index} className="tag">#{tag}</span>
+ <span key={tag}   className="tag">#{tag}</span>
```

Usar o valor estável da tag como key é mais correto que o índice do array.

### 🔧 Correções nos GitHub Actions

#### 7. Versões inválidas de Actions corrigidas

**Contexto:** O Dependabot falhava ao tentar resolver as Actions com versões futuras (`@v6`, `@v7`) que ainda não existiam, e acabava consultando o registry privado `npm.pkg.github.com` sem autenticação.

**Arquivos:** `.github/workflows/build.yml`, `.github/workflows/deploy.yml`

| Action                          | Antes | Depois |
| ------------------------------- | ----- | ------ |
| `actions/checkout`              | `@v7` | `@v4`  |
| `pnpm/action-setup`             | `@v6` | `@v4`  |
| `actions/setup-node`            | `@v6` | `@v4`  |
| `actions/configure-pages`       | `@v6` | `@v5`  |
| `actions/cache`                 | `@v6` | `@v4`  |
| `actions/upload-pages-artifact` | `@v5` | `@v3`  |
| `actions/deploy-pages`          | `@v5` | `@v4`  |

---

#### 8. `next.config.ts` — Corrigido workspace root do Turbopack

**Problema:** O Next.js detectava um `package-lock.json` em `C:\Users\jocil\` (fora do projeto) e confundia o workspace root, emitindo um warning a cada build.

**Correção:**

```ts
turbopack: {
  root: __dirname,
},
```
